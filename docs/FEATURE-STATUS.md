## How to Read This

- **WORKING** — Tested in production, confirmed functional
- **PARTIAL** — Some features work, some don't, or needs config to activate

## Content Pipeline

| Feature | Status | Notes |
|---------|--------|-------|
| Topic Discovery | WORKING | HackerNews, Dev.to, DuckDuckGo. Sources/keywords are hardcoded to AI/ML, gaming, PC hardware niches. Not configurable without code changes (#157). |
| Web Research | WORKING | Serper API for deep search. Falls back to DuckDuckGo if no API key. |
| Content Generation | WORKING | Ollama local inference. Zero API costs. Multiple model support. |
| Quality Scoring | WORKING | 7-dimension evaluation. Programmatic anti-hallucination validator with 30+ pattern categories. |
| Inline Images | WORKING | Image Decision Agent (phi4:14b) plans placement. SDXL generates on GPU (~1.5s each). Falls back to Pexels stock photos. Uploaded to R2 CDN. |
| Featured Images | WORKING | SDXL generated via external server. Falls back to Pexels. Uploaded to R2. |
| Cross-Model QA | WORKING | Adversarial review (writer and critic are different models). Fallback chain if primary returns empty. |
| SEO Metadata | WORKING | Title, description, keywords auto-generated by LLM. |
| Podcast Generation | WORKING | Microsoft Edge TTS (free, local). MP3 with intro/body/outro. 5 voice options. |
| Training Data Capture | WORKING | Post embeddings stored in pgvector for RAG/learning. |
| Auto-Curation | WORKING | Posts below quality threshold (80) auto-rejected. Above threshold → awaiting human approval. |

### Known Issues
- Topic discovery keywords hardcoded to 3 niches — users in other niches get irrelevant topics (#157)
- Quality scores typically 72-82 range — good but not consistently great
- qwen3:8b returns empty on long prompts (use phi4:14b for image decisions)
- glm-4.7-5090 sometimes returns empty QA reviews (gemma3:27b fallback handles this)

### What Distribution Actually Looks Like Today
1. Post publishes → static JSON pushed to R2 → your Next.js frontend updates
2. Podcast MP3 generated → served via RSS feed
3. Social post text generated → sent to Telegram/Discord notification → you manually post to X/LinkedIn
4. Newsletter → sends to subscribers via Resend (just enabled, needs production testing)
5. Dev.to → skipped (no API key)
