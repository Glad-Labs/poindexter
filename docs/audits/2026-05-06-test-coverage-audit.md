## The 7 currently-failing cost_lookup tests are real, not stale

`tests/unit/services/test_cost_lookup.py` has 7 tests that **fail when run alongside `test_litellm_langfuse_callback.py`** but **pass when run alone**. Root cause: `test_litellm_langfuse_callback.py:35` mutates `sys.modules` directly (`sys.modules.setdefault("litellm", _litellm_stub)`) without monkeypatch teardown. `test_cost_lookup.py` then imports the real LiteLLM lookup, which does `import litellm` — gets the lingering MagicMock — and `getattr(litellm, "model_cost", None) or {}` returns a MagicMock that responds truthy to `.get(model)` and returns `1.0` for `input_cost_per_token`. The math compounds: `(1000/1000) * 1.0 * 1000 = 1000.0` per 1K tokens.

This is fixable in five lines (swap `sys.modules.setdefault` for `monkeypatch.setitem(sys.modules, ...)` in the autouse fixture, restoring the real module on teardown). The fix is in this PR's quick-win batch.

## Recommended test additions, ordered

Priority based on `(load-bearing × uncovered LOC × silent-breakage probability)`. The first three are in this PR.

### Quick wins (this PR)

1. **`tests/unit/services/test_litellm_provider_complete.py`** — exercise `LiteLLMProvider.complete()` end-to-end. Mocks `litellm.acompletion` to return shaped responses, verifies the provider:
   - Resolves bare model names (`gemma3:27b` → `ollama/gemma3:27b`).
   - Leaves namespaced names alone (`anthropic/claude-haiku-4-5`).
   - Threads `_provider_config` overrides into `_configure_from`.
   - Surfaces `response_cost` from the LiteLLM response into `Completion.raw["response_cost"]` so cost_logs can read it without re-deriving.
   - Re-raises on `acompletion` exceptions (no silent fallback — per `feedback_no_silent_defaults`).
   - Catches: regression where `Completion.raw` drops `response_cost` and cost tracking goes silently to zero on real cloud calls.

2. **`tests/unit/services/test_litellm_provider_stream_embed.py`** — exercise `stream()` and `embed()`.
   - `stream` yields `Token` objects from each chunk; `finish_reason` set on the terminal token.
   - `embed` unpacks `response.data[0].embedding` and `response.data[0]["embedding"]` shapes equivalently.
   - Catches: regression where streaming breaks token attribution (writer pipeline shows zero tokens because `delta.content` returns None and we don't fall through).

3. **`tests/unit/services/test_cost_lookup.py` — fix the test-pollution bug** in `test_litellm_langfuse_callback.py`. Swap `sys.modules.setdefault` for `monkeypatch.setitem(sys.modules, ...)` so the real `litellm` is restored on teardown. Unblocks 7 currently-failing tests AND closes the cost_lookup gap.
   - Catches: the failure that made me investigate this — the LiteLLM langfuse-callback test file blocks unrelated cost-lookup tests in CI when collected in any order other than alphabetical-after.

4. **`tests/unit/services/test_prompt_manager_langfuse_fallthrough.py`** — exercise the Langfuse fall-through chain in `prompt_manager.py`. Covers:
   - Langfuse import failure → `_init_langfuse_client` returns None, falls through to YAML.
   - SiteConfig missing required Langfuse keys → returns None, logs once at INFO.
   - `_fetch_from_langfuse` exception → demoted to debug log, falls through.
   - Catches: silent regression where Langfuse credential rotation breaks fetches and `_langfuse_enabled = False` flips to False, but no operator-visible warning fires.

### Deferred (skip in this PR — write up follow-up if revisited)

5. **`template_runner.make_stage_node` error paths** — the timeout / exception / halt branches. Real value, but the LangGraph + StageRunner integration tests in `tests/integration/` already exercise the happy paths at the workflow level. Per the task brief: don't pursue coverage % vanity. Defer until template_runner becomes the default orchestrator (gated on #356).

6. **`publish_service` revalidation + RSS regen failure paths** — the file is well-covered on success paths. Failure paths involve real HTTP / file-IO mocks that mostly test the mocking layer, not the logic. Defer.

7. **`multi_model_qa` image-relevance + rendered-preview gates** — both are opt-in (`qa_vision_gate_enabled` / `qa_preview_screenshot_enabled` default false). Until they ship as on-by-default, regression risk is low.

8. **`workflow_executor` failure paths** — being replaced by `template_runner.py` per #356. Don't sink test investment into a deprecation target.

## What's NOT worth covering

To prevent re-running this audit on these files:

- **`services/voice_agent_*.py` (4 files, 913 LOC)** — voice/LiveKit code, exercised only by voice integration tests. Out of pipeline hot path.
- **`services/topic_discovery.py` (390 LOC), `services/topic_sources/*` (430 LOC across 6 files)** — already integration-tested via `test_topic_*.py` files (12 of them, ~1,500 LOC). Branch coverage is fine; the 0% in this slice is an artifact of NOT running those tests in this audit run.
- **`services/migrations/*` (161 files)** — smoke-tested in CI via `scripts/ci/migrations_smoke.py`. Per task brief: LOW priority.
- **`services/jobs/*` (28 files)** — CLI-style cron jobs. Each has its own targeted unit test file. Low ROI to add cross-file integration tests.
- **`services/video_providers/*`, `services/audio_gen_providers/*`, `services/image_providers/*`** — provider plugins. Each has its own test file. New providers should add a test, not be retroactively swept.
- **`services/voice_agent_livekit.py` (465 LOC)** — actively-iterating voice agent. Unit-test investment here churns with the upstream LiveKit SDK.
- **CLI entry points (`poindexter/cli/*`)** — wired to subcommand integration tests. Low silent-breakage risk: a broken CLI surfaces immediately on the next operator command.
- **`services/sync_service.py`, `services/static_export_service.py`** — invoked by Vercel deploy hook, exercised end-to-end on every push to main. Failure is loud (deploy fails).

## Methodology footnotes

- Test files run: 31, picked by intersection of (load-bearing service list) ∪ (anti-hallucination layer) ∪ (cost surface).
- Coverage tool: `pytest-cov 4.1.0` + `coverage 7.13.5`.
- Out-of-scope: `tests/integration/` (DB-backed) and `tests/integration_db/` (live Postgres harness from #21). Adding integration coverage requires the harness, not unit-test additions.
- The audit deliberately did NOT run `pytest tests/unit/ -q --cov` over all 7,900 tests — that pulls in singleton/sys.modules cross-pollination from unrelated tests (the cost_lookup failure is one example; there are likely more lurking) and the per-file numbers stop being trustworthy. Per-test-file coverage runs are noisier-by-design but produce honest per-file ownership signal.

## Honest read

The codebase is **appropriately tested** for a 7,900-test suite. The 18 load-bearing services average 80% line coverage; the highest-leverage anti-hallucination + cost services (`cost_guard`, `qa_gates_db`, `site_config`, `settings_service`) are at 96-100%. The two genuine gaps:

1. **`LiteLLMProvider` class is post-#372 code with no tests on its `complete`/`stream`/`embed` methods.** This is the production-cutover provider. Closing it is a small focused PR (this one).
2. **One test file pollutes `sys.modules["litellm"]`** and breaks 7 cost-lookup tests downstream. Five-line fix.

Net new test code from this audit: ~250 LOC. Coverage improvement: not the point. Regression safety improvement: meaningful, scoped, and shippable today.
