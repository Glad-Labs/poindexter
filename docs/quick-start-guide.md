### Should Work (Untested)

| Setup | Notes |
|-------|-------|
| **Linux + NVIDIA GPU** | Actually easier than Windows (native Docker, no Docker Desktop needed) |
| **RTX 4090 (24GB)** | Sweet spot for most users. All models fit. |
| **RTX 4070/5070 (12GB)** | Works with smaller models. Use qwen3:8b for writing, skip 27B+ models. |
| **RTX 3060+ (8GB)** | Minimum for GPU inference. Use 8B models only. SDXL image gen still works. |

### Minimum Requirements

- **Docker Desktop** (or Docker Engine on Linux)
- **8GB+ GPU VRAM** (for AI inference) — or CPU mode (slow)
- **16GB RAM** minimum, 32GB recommended
- **50GB free disk** (models + generated content + Docker images)

## 1. Install Prerequisites (5 min)

### Docker
- **Windows/macOS:** [Docker Desktop](https://docker.com/products/docker-desktop)
- **Linux:** `curl -fsSL https://get.docker.com | sh`

### Ollama (Local AI Inference)
- **All platforms:** [ollama.com](https://ollama.com)
- After install, verify: `ollama --version`
- Start the server: `ollama serve` (runs in background)

### Git + Node.js (for frontend, optional)
- [Git](https://git-scm.com)
- [Node.js 22+](https://nodejs.org) (only needed if you want to run the web frontend)

## 2. Clone + Bootstrap (5 min)

```bash
git clone https://github.com/Glad-Labs/glad-labs-codebase.git
cd glad-labs-codebase

# One-command setup: creates config, starts DB, pulls models, seeds settings
bash scripts/bootstrap.sh
```

The bootstrap script:
- Creates `.env.local` with auto-generated secrets (API token, passwords)
- Starts PostgreSQL via Docker
- Installs Python + Node.js dependencies
- Seeds 185 configuration settings with working defaults
- Pulls minimum AI models (qwen3:8b, gemma3:27b, nomic-embed-text)
- Detects your hardware and recommends models

### What gets auto-generated

| Setting | Generated By |
|---------|-------------|
| `API_TOKEN` | Random 32-byte token |
| `GRAFANA_PASSWORD` | Random 16-byte password |
| `LOCAL_POSTGRES_PASSWORD` | Random 16-byte password |
| `PGADMIN_PASSWORD` | Random 16-byte password |

These are written to `.env.local`. The file is gitignored — your secrets stay local.

## 3. Start the Engine (2 min)


| `gitea` | Self-hosted Git (optional) | 3001 |
| `woodpecker` | CI/CD (optional) | 8280 |
| `headscale` | VPN (optional) | 8080 |
| `pgadmin` | Database GUI (optional) | 5050 |

## 4. Your First Post (2 min)

```bash
# Get your API token from .env.local
TOKEN=$(grep API_TOKEN .env.local | cut -d= -f2)

# Create a content task
curl -X POST http://localhost:8002/api/tasks \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "topic": "Why every developer needs a personal AI factory",
    "task_type": "blog_post",
    "category": "technology"
  }'
```

The pipeline runs automatically:

| Stage | What Happens | Time |
|-------|-------------|------|
| 1. Research | Web search + source verification | ~10s |
| 2. Draft | Local LLM generates 1500-word article | ~30-60s |
| 3. Quality Score | Pattern-based scoring on 7 dimensions | ~1s |
| 4. URL Validation | Verifies all links actually exist | ~5s |
| 5. Image Planning | AI decides what images each section needs | ~7s |
| 6. Image Generation | SDXL generates inline images on your GPU | ~3s/image |
| 7. Cross-Model QA | Different LLM reviews the draft | ~15s |
| 8. SEO Metadata | Title, description, keywords generated | ~5s |
| 9. Media Scripts | Podcast script + video scenes generated | ~30s |
| 10. Finalize | Auto-publish if score >= 75, or queue for review | instant |

**Total: ~2-3 minutes per post.**

Watch progress in Grafana: http://localhost:3000 → Pipeline dashboard.

## 5. Configuration — Everything Is in the Database

Every setting lives in the `app_settings` table. Change a value, the pipeline uses it on the next run. No code changes, no restarts, no deploys.

### View all settings
```bash
curl http://localhost:8002/api/settings \
  -H "Authorization: Bearer $TOKEN"
```

### Change a setting
```bash
# Raise the quality bar
curl -X PUT http://localhost:8002/api/settings/qa_overall_score_threshold \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"value": "80"}'
```

### Key settings to customize first

| Setting | Default | What It Does |
|---------|---------|-------------|
| `site_name` | My Content Site | Your brand name (appears in RSS, metadata) |
| `site_url` | http://localhost:3000 | Your production URL |
| `site_domain` | localhost:3000 | Your domain for link validation |
| `qa_overall_score_threshold` | 70 | Minimum score to pass QA (0-100) |
| `auto_publish_threshold` | 0 | Score to auto-publish (0 = manual approval only) |
| `daily_spend_limit` | 2.0 | Max daily cloud API spend in USD |
| `pipeline_writer_model` | ollama/qwen3.5:35b | LLM for content generation |
| `pipeline_critic_model` | ollama/gemma3:27b | LLM for QA reviews |
| `model_role_image_decision` | ollama/phi4:14b | LLM for image placement decisions |
| `enable_featured_image` | true | Generate/search featured images |
| `pexels_api_key` | (empty) | Free API key from pexels.com for stock photos |

### Choosing your models

Your GPU VRAM determines which models you can run:

| VRAM | Writer Model | QA Model | Image Decision |
|------|-------------|----------|----------------|
| 8GB | qwen3:8b | qwen3:8b | qwen3:8b |
| 12GB | qwen3:8b | gemma3:12b | phi4:14b (swap) |
| 16GB | gemma3:27b (Q4) | gemma3:12b | phi4:14b |
| 24GB | qwen3:30b | gemma3:27b | phi4:14b |
| 32GB+ | glm-4.7-5090 | gemma3:27b | phi4:14b |

Set models via:
```sql
UPDATE app_settings SET value = 'ollama/qwen3:8b'
WHERE key = 'pipeline_writer_model';
```

## 6. Image Pipeline

The engine generates AI images for your posts using Stable Diffusion XL on your GPU.

### How it works
1. **Image Decision Agent** (phi4:14b) reads your article and decides which sections need images
2. **SDXL server** generates each image in ~1 second on your GPU
3. Images are uploaded to R2 CDN (or saved locally)
4. Markdown image tags are inserted at the planned locations
5. A featured/hero image is generated separately

### SDXL Server Setup

The SDXL server runs as a standalone FastAPI service:

```bash
# Start the SDXL server (uses your GPU)
python scripts/start-sdxl-server.py
```

It auto-unloads after 1 minute of idle time to free VRAM for Ollama. The GPU scheduler handles switching between inference and image generation automatically.

### No GPU? Pexels Fallback

If SDXL isn't available, the engine falls back to Pexels stock photos (free API). Get a key at [pexels.com/api](https://www.pexels.com/api/) and set it:

```sql
UPDATE app_settings SET value = 'your-key-here'
WHERE key = 'pexels_api_key';
```

## 7. Static Export — Connect Any Frontend

When a post publishes, the engine pushes static JSON to your configured storage. Any frontend can read these files — zero API dependency.

### What gets exported

```
your-bucket/
  static/
    posts/
      index.json          — all posts (paginated)
      my-first-post.json  — individual post with full content
    feed.json             — JSON Feed 1.1
    categories.json       — all categories
    authors.json          — author profiles
    sitemap.json          — sitemap data
```

### Cloudflare R2 Setup (Recommended — Free)

1. Create an R2 bucket in your Cloudflare dashboard
2. Enable public access on the bucket
3. Create an API token with R2 read/write permissions
4. Configure in the database:

```sql
UPDATE app_settings SET value = 'your-access-key' WHERE key = 'r2_access_key_id';
UPDATE app_settings SET value = 'your-secret-key' WHERE key = 'r2_secret_access_key';
UPDATE app_settings SET value = 'https://your-account-id.r2.cloudflarestorage.com' WHERE key = 'r2_endpoint';
UPDATE app_settings SET value = 'your-bucket-name' WHERE key = 'r2_bucket';
UPDATE app_settings SET value = 'https://pub-xxx.r2.dev' WHERE key = 'r2_public_url';
```

### Connect your frontend

```javascript
// Any framework — Next.js, Hugo, Astro, plain HTML
const res = await fetch('https://your-bucket.r2.dev/static/posts/index.json');
const { posts } = await res.json();

// Each post has: title, content (markdown), excerpt, slug, featured_image_url,
// published_at, category, tags, seo_title, seo_description, reading_time_minutes
```

No API server needed. No authentication. Just static JSON files on a CDN.

Get notified when things break:

1. Create a Telegram bot via [@BotFather](https://t.me/BotFather)
2. Get your chat ID (message the bot, check the API)
3. Configure:

```sql
UPDATE app_settings SET value = 'your-chat-id' WHERE key = 'telegram_chat_id';
```

Alerts fire on: stuck tasks, high error rate, GPU temperature, worker offline.

## 9. Autonomous Operation — The Brain Daemon

The brain daemon runs 24/7, monitoring your system and self-healing failures:

- **Health checks** every 5 minutes (site, API, database, Ollama, GPU)
- **Auto-restart** crashed containers and services
- **Alert triage** — critical issues go to Telegram, non-critical logged
- **Knowledge graph** — maintains a self-updating map of your system state

It's a separate Docker container (`gladlabs-brain-daemon`) that starts automatically with the stack.

### The idle worker

When no tasks are pending, the worker doesn't sit idle. It:
- Discovers trending topics from HackerNews, Dev.to, Reddit
- Checks existing content for broken links
- Cleans up stale tasks
- Syncs database state

This means your pipeline generates content continuously without manual intervention.

## 10. Troubleshooting

### Pipeline not generating content
```bash
# Check Ollama is running
curl http://localhost:11434/api/tags

# Check worker health
curl http://localhost:8002/api/health

# Check for throttling (approval queue full)
docker logs gladlabs-worker --tail 20 | grep THROTTLE
```

### Posts stuck at "awaiting_approval"
- The approval queue has a limit (default: 3). Approve or reject pending posts.
- Lower the threshold: `UPDATE app_settings SET value = '0' WHERE key = 'auto_publish_threshold';` (0 = auto-approve everything above qa_overall_score_threshold)

### Images not generating
- Check SDXL server: `curl http://localhost:9836/health`
- If no GPU/SDXL, set a Pexels API key for stock photo fallback
- Check the image decision model is pulled: `ollama list | grep phi4`

### "Model not found" errors
- List available models: `ollama list`
- Pull what you need: `ollama pull qwen3:8b`
- Check what the pipeline expects: `SELECT key, value FROM app_settings WHERE key LIKE 'pipeline_%_model' OR key LIKE 'model_role_%';`

### Database connection failed
- Check PostgreSQL container: `docker ps | grep postgres`
- Verify password: compare `LOCAL_POSTGRES_PASSWORD` in `.env.local` with what Docker was started with
- The DB runs on port 5433 (not default 5432) to avoid conflicts

### Everything else
- Run the health check: `bash scripts/verify-setup.sh`
- Check worker logs: `docker logs gladlabs-worker --tail 50`
- Check brain daemon logs: `docker logs gladlabs-brain-daemon --tail 50`
