#### New features

- **Niche-aware topic discovery + RAG writer modes.** Poindexter now treats your install as one of many possible niches instead of defaulting to an evergreen-tech voice — run the same engine on real estate, fashion, finance, or any other editorial voice. Topic batches replace one-off proposals: a ranked top-5 of candidates is presented for you to rank, edit, and resolve into a content task, with near-misses carrying forward across cycles. The writer ships with four configurable RAG modes (topic-only, citation-budget, story-spine, two-pass) so you can dial how aggressively each niche grounds drafts in your own corpus. New CLI (`poindexter topics …`, `poindexter niche …`) and matching MCP tools let Claude Code drive the whole gate end-to-end. ([#278](https://github.com/Glad-Labs/poindexter/pull/278))
- **Code-block density gate for tech posts.** Multi-model QA now warns when a post tagged as technical ships without enough fenced code blocks to actually demonstrate the technique. It's a soft check, not a hard block — prose-only tech posts (architecture overviews, postmortems) still pass at your discretion. Five settings let you tune or disable the gate per install. ([#274](https://github.com/Glad-Labs/poindexter/pull/274))
- **Migration drift in `/api/health`.** The health endpoint now reports the latest applied migration, how many are pending, and flips overall status to `degraded` if any are unapplied. Uptime Kuma, the brain daemon, and any external monitor will trip automatically when a deploy lands code without its schema. ([#270](https://github.com/Glad-Labs/poindexter/pull/270))
- **Operator URL drift probe.** The brain daemon now checks operator-facing URLs and VPN IPs every 15 minutes — Grafana dashboards, Tailscale hosts, and `*_url` settings — and sends a single Telegram notification with the exact fix when something goes stale. Stale dashboard links and renumbered VPN hosts now surface as alerts instead of silent dead-ends. See the [brain probe README](brain/README.md). ([#214](https://github.com/Glad-Labs/poindexter/issues/214), [#256](https://github.com/Glad-Labs/poindexter/pull/256))
- **Three-layer anti-hallucination architecture documented.** A new design guide explains how Poindexter prevents fabricated facts at the writer, citation, and QA layers. See [docs/architecture/anti-hallucination](docs/architecture/anti-hallucination.md).

#### Updates

- **Project rebrand: Glad Labs Engine → Poindexter.** The product is now Poindexter (Glad Labs LLC remains the legal entity). For existing installs:
  - Customer-facing env vars are renamed (`GLADLABS_KEY` → `POINDEXTER_KEY`, `GLADLABS_API_URL` → `POINDEXTER_API_URL`, etc.). Old names still work as a fallback.
  - The data directory moves from `~/.gladlabs` to `~/.poindexter` — move it once before restarting the worker.
  - Customer-facing containers, the Postgres database/role, and the Prometheus metric prefix all switch from `gladlabs*` to `poindexter*`.
  - Public GitHub repo: `glad-labs-engine` → `poindexter`.
- **Niche-configurable topic filters.** Two more topic-discovery filters that assumed a tech editorial voice are now overridable per install. Set `topic_discovery_news_patterns` to control which titles get rejected as news, and `topic_discovery_category_searches` to define your own classification buckets. Empty defaults preserve the built-in tech behavior, so existing deployments are unchanged until you opt in. ([#276](https://github.com/Glad-Labs/poindexter/pull/276))
- **MCP server split into public + private.** The single Claude Code MCP entry is now two servers. `poindexter` (public, ships with the product) carries the full content-pipeline surface — create/approve/reject/publish posts, settings, memory, audit log, brain knowledge, and health. A private `gladlabs` operator layer holds personal integrations and is excluded from the public mirror.
- **First-install hardening.** `bootstrap.sh` now generates a missing `WOODPECKER_SECRET`, hard-fails on a missing entropy source instead of silently producing weak secrets, seeds writer/critic model defaults that match the models the script actually pulls, and prints the same `docker compose` commands as the README. The README also now flags that Windows customers need Git Bash or WSL.
- **Per-probe traces in the brain daemon.** Each health probe now runs inside its own `brain.probe.<name>` child span with `probe.name`, `probe.duration_s`, and `probe.ok` attributes. Slow or failing probes show up individually in Tempo flame graphs instead of aggregating under a single parent span, so it's much easier to tell which probe is the source of a long brain run. No-op when the OpenTelemetry SDK isn't installed. ([#291](https://github.com/Glad-Labs/poindexter/pull/291))
- **Five more content-agent tunables moved to `app_settings`.** `SERPER_API_KEY`, `LOCAL_LLM_API_URL`, `LOCAL_LLM_MODEL_NAME`, `MAX_LOG_SIZE_MB`, and `MAX_LOG_BACKUP_COUNT` are now configured through the database-backed settings store (Serper key stored as a secret), so you can change them on a running install without editing env files or redeploying. Existing env-var values still work as a fallback, so no action is required for current deployments. ([#300](https://github.com/Glad-Labs/poindexter/pull/300))

#### Bug fixes

- **MCP `topics_propose` no longer crashes on launch.** A missing runtime dependency made the entire topic-gate flow (`topics_propose`, `topics_show`, `topics_reject`, `topics_approve`) fail at import. Reinstall or upgrade the MCP server to pick up the fix. ([#258](https://github.com/Glad-Labs/poindexter/issues/258))
- **MCP tool errors are now actionable.** Failures used to surface as bare `… failed:` messages with no exception type and no correlation id. Errors now include the exception class and a short `rid=` request id you can quote in a bug report; the full traceback is logged server-side under the same id. ([#259](https://github.com/Glad-Labs/poindexter/issues/259))
- **`get_setting` / `set_setting` accept the `category/key` form** that `list_settings` prints (e.g. `site/public_site_url`). Copy-paste from `list_settings` output now works without manually stripping the prefix. ([#253](https://github.com/Glad-Labs/poindexter/issues/253))
- **Audit log no longer floods with errors during dry-run mode.** Expected halts under `pipeline_dry_run_mode` were logged at `error` severity, drowning the 24h error count and hiding real failures. Dry-run halts now log at `info` under a dedicated event type; real errors (model timeouts, DB issues) still log at `error`. ([#260](https://github.com/Glad-Labs/poindexter/issues/260))
- **RSS feed regeneration is now collision-safe.** Concurrent publishes could previously race on a shared scratch file and produce a truncated or empty podcast/video feed. Each regen now uses its own isolated working file, so feeds stay intact under load. ([#284](https://github.com/Glad-Labs/poindexter/issues/284))
- **Windows installs no longer fail on the `profiling` extras.** `poetry install --extras profiling` on Windows used to fall through to a Rust source build for `pyroscope-io` (which has no Windows wheels) and crash the install. The dependency is now scoped to non-Windows platforms, so Windows users can pull the profiling extras cleanly while the Linux worker container still installs it normally. ([#224](https://github.com/Glad-Labs/poindexter/issues/224), [#297](https://github.com/Glad-Labs/poindexter/pull/297))
- **Fresh-database installs now apply every migration cleanly.** New deployments that bootstrapped against a brand-new Postgres (no `init.sql` pre-seed) hit failures partway through the migration chain — 10 of 95 migrations would error or skip, leaving the schema in an undefined state. The chain is now self-contained: `app_settings.is_active` is created up front, late-migration views and tables guard against pre-niche-pivot vs. post-niche-pivot ordering, and a number collision on slot `0093` was resolved by renumbering `object_stores` to `0117`. Fresh-DB CI smoke now passes end-to-end, so first-time installs reach a working schema without manual intervention. ([#86](https://github.com/Glad-Labs/poindexter/pull/86), [#87](https://github.com/Glad-Labs/poindexter/pull/87))

#### Known issues

- `bootstrap.sh` still has rough edges around silent failure cleanup, version checks, and schema drift. A larger rewrite is planned.
